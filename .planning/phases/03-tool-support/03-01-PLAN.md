    #### _convert_tool_format(tool: dict) -> dict（D-08, D-09）

    ```python
    def _convert_tool_format(tool: dict) -> dict:
        """扁平工具格式 → 嵌套格式转换。

        处理:
        - 已嵌套（有 function 键）→ 原样通过
        - 未知 type → WARNING + 原样透传
        - function type 扁平 → {type, function: {name, desc, params, strict}}
        """
        # 1. 若已有 "function" 键 → 直接返回（D-08 已嵌套处理）
        if "function" in tool:
            return tool

        # 2. 未知 type 记录 WARNING 并原样透传
        tool_type = tool.get("type", "")
        if tool_type != "function":
            logger.warning("[CODEX] unknown tool type: %s, passing through", tool_type)
            return tool

        # 3. function type 扁平格式 → 嵌套包装
        #    需要移到 function 下的键: name, description, parameters, strict
        function_fields = {}
        for key in ("name", "description", "parameters", "strict"):
            if key in tool:
                function_fields[key] = tool.pop(key)

        # 4. 构建嵌套格式（保留 type + 其他可能的顶层字段）
        tool["function"] = function_fields
        return tool
    ```

    关键约束:
    - 使用 `tool.get("type", "")` 避免 None/缺失 type 时的 AttributeError
    - `tool.pop(key)` 移除顶层扁平字段（PATTERNS.md 确认模式）
    - 严格检查 `"function" in tool` 而不是假设嵌套格式
    - 使用 `%s` 风格日志（不是 f-string）

    #### _clean_schema(schema: dict) -> dict（D-04, D-05, D-06）

    ```python
    def _clean_schema(schema: dict) -> dict:
        """递归剥离 DeepSeek 不兼容的 JSON Schema 字段。

        在 convert_tools 的 deepcopy 保护下安全地原地修改输入（效率优化）。
        """
        # 1. 剥离不兼容字段（D-05 精确字段清单）
        for key in REMOVED_KEYS:
            schema.pop(key, None)

        # 2. 移除空 enum 数组
        if "enum" in schema and isinstance(schema["enum"], list) and len(schema["enum"]) == 0:
            del schema["enum"]

        # 3. 递归: properties（D-06 必须处理的路径）
        if "properties" in schema and isinstance(schema["properties"], dict):
            for sub_schema in schema["properties"].values():
                if isinstance(sub_schema, dict):
                    _clean_schema(sub_schema)

        # 4. 递归: $defs
        if "$defs" in schema and isinstance(schema["$defs"], dict):
            for sub_schema in schema["$defs"].values():
                if isinstance(sub_schema, dict):
                    _clean_schema(sub_schema)

        # 5. 递归: anyOf
        if "anyOf" in schema and isinstance(schema["anyOf"], list):
            for item in schema["anyOf"]:
                if isinstance(item, dict):
                    _clean_schema(item)

        # 6. 递归: items
        if "items" in schema and isinstance(schema["items"], dict):
            _clean_schema(schema["items"])

        return schema
    ```

    关键约束:
    - 每一个 `pop` 都用 `None` 默认值避免 KeyError
    - 所有类型检查用 `isinstance()`（与 translate.py 一致）
    - 所有递归路径用 guard 保护（isinstance 检查），不假设 dict 类型
    - 返回 schema 以便链式调用（非必须但遵循 translate.py convention）
    - 使用 `del` 而非 `pop` 删除 enum — 因为需要先确认它存在且为空

    ### 5. 公共 API 函数（D-01 主入口）

    分隔注释: `# ---- 公开 API ----`

    ```python
    def convert_tools(tools: list[dict]) -> list[dict]:
        """Responses API 工具列表 → DeepSeek Chat 兼容格式。

        Pure function: 不修改输入（deepcopy 保护）。

        Args:
            tools: 工具定义字典列表（不会被修改）。

        Returns:
            格式转换 + schema 清理后的工具定义列表。

        处理步骤:
        1. 深拷贝输入（None → []）
        2. 对每个工具调用 _convert_tool_format()
        3. 若有 parameters 且为 dict → 调用 _clean_schema()
        4. 若有 parameters 且非 dict → 抛出 ValueError
        5. 返回处理后的列表
        """
        if not tools:
            return []

        result = copy.deepcopy(tools)

        for i, tool in enumerate(result):
            if not isinstance(tool, dict):
                logger.warning("[CODEX] non-dict tool at index %d, skipping", i)
                continue

            # 格式转换
            converted = _convert_tool_format(tool)

            # Schema 修复（在 function 内的 parameters 或顶层的 parameters）
            params = converted.get("parameters")
            # 如果已嵌套，parameters 在 function 里
            if "function" in converted:
                params = converted["function"].get("parameters")

            if params is not None:
                if isinstance(params, dict):
                    _clean_schema(params)
                elif isinstance(params, list):
                    # D-07: list 类型也视为无效 — schema 必须是 dict
                    tool_name = converted.get("function", {}).get("name", "?") if "function" in converted else converted.get("name", "?")
                    raise ValueError(
                        f"Invalid tool parameters schema for '{tool_name}': "
                        f"expected dict, got list"
                    )
                else:
                    # D-07: 非 dict 非 None 的 parameters → 抛出异常（Pitfall 4）
                    tool_name = converted.get("function", {}).get("name", "?") if "function" in converted else converted.get("name", "?")
                    raise ValueError(
                        f"Invalid tool parameters schema for '{tool_name}': "
                        f"expected dict, got {type(params).__name__}"
                    )

        return result
    ```

    关键约束:
    - `if not tools: return []` — 处理 None 和空列表（D-09）
    - 使用 `copy.deepcopy(tools)` — 输入保护（PATTERNS.md 模式）
    - `_clean_schema(params)` 在转换后的 function 内 parameters 上调用
    - 非 dict parameters 抛出 `ValueError`（D-07），包含工具名 + 类型名
    - WARNING 日志使用 `%s` 风格和 `[CODEX]` 前缀

    ## 注意事项

    ### D-04（只剥离不补全）
    - 绝对不要添加 `required`、`additionalProperties: false` 或其他字段
    - 只做减法（字段剥离），不做加法（字段补全）

    ### D-06（递归清理）
    - 清理深度不限，以下路径都必须递归:
      - `properties.*` — 每个子属性的 schema
      - `$defs.*` — 每个引用的 schema
      - `anyOf[*]` — 数组中的每个分支 schema
      - `items` — 数组项的 schema
    - 每个递归入口都要 isinstance 检查，避免 AttributeError

    ### Parameters 位置
    - `_convert_tool_format()` 转换后，flat 格式的 parameters 被移到 `tool["function"]["parameters"]`
    - `_clean_schema()` 总是在 `function["parameters"]` 上调用
    - 但如果工具已经是嵌套格式（已有 function 键），parameters 已在 function 内部
    - 代码中的 params 定位逻辑需要处理两种情况

    ### 日志前缀
    - 所有日志使用 `[CODEX]` 前缀
    - logger.warning 用于: 未知 type（D-08）、非 dict tool 跳过
    - logger.debug 可选用: 转换成功（非必须）

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| convert_tools input entry | 外部工具定义 dict 列表进入转换函数，包含嵌套 JSON Schema |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | convert_tools entry | mitigate | Input is deep-copied (D-03 protection). Non-dict items in tools list are skipped with WARNING. All dict traversal uses isinstance() guards — no eval or unsafe parsing. |
| T-03-02 | Denial of Service | _clean_schema recursion | accept | Deeply nested schemas (depth > 1000) from malicious input could cause stack overflow. Mitigation: pytest default recursion limit (1000) mirrors Python stack limit (typically 1000). No recursive exploit vector distinguishable from legitimate deep schemas. Accept risk for v1. |

**Security rationale:** Phase 3 is a pure data transformation module with no HTTP handling, no authentication, no file I/O, no network access, and no database operations. All inputs are in-memory Python dicts obtained from the already-parsed request body. The only threat vector is malformed input causing exceptions, which is handled by D-07 (raise ValueError for invalid schemas) and isinstance() guards on all recursive paths. Schema stripping removes metadata fields (default/examples) that could contain sensitive default values — an incidental security benefit.
</threat_model>
