<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env-var → config.py | Environment variables provide untrusted input to module-level configuration loading. CODEX_MODEL_MAP is JSON that must be parsed defensively. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-01 | Tampering | CODEX_MODEL_MAP env var parsing in _parse_model_map() | medium | mitigate | Catch `json.JSONDecodeError` with try/except; log warning via logger.warning; return empty dict `{}`. Also validate parsed result is `dict` type, reject non-dict values with warning. |
| T-01-02 | Spoofing | resolve_model return value | low | accept | Returns only string values from env vars or hardcoded defaults. No user-controlled string interpolation or code injection path. Model names are labels, not executables. |
| T-01-03 | Information Disclosure | Debug logging in resolve_model() | low | accept | Logger at DEBUG level only (inactive by default). Model names logged are already user-provided (no secrets). No PII, credentials, or tokens logged. |
| T-01-04 | Denial of Service | Extremely large CODEX_MODEL_MAP JSON | low | accept | JSON parsing is stdlib (no decompression bomb risk). Dict size is bounded by user's env var string length (~32KB typical max). Iteration in resolve_model is O(n) where n < 100 in practice. |

## Key Mitigations

- **Parsing safety**: `_parse_model_map()` wraps `json.loads()` in try/except `json.JSONDecodeError`. Additionally validates the parsed result is `dict` type via `isinstance()`. Both failure paths log warnings and return empty dict `{}`.
- **No eval/exec**: All configuration parsing uses `json.loads()` only. No `eval()`, `exec()`, or dynamic import that could execute untrusted code.
- **Safe defaults**: Every env var has a hardcoded default value. If env var is unset, empty, or malformed, the system degrades gracefully rather than crashing.
