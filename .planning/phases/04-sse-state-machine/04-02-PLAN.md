    === 文件结构 ===

    ```python
    """dsv4-cc-proxy SSE 状态机单元测试。

    覆盖: 基础文本流、推理流、推理->文本转换、工具调用流、
          多工具并行、文本->工具转换、完整生命周期、边界情况。

    运行: python3 -m pytest tests/test_sse.py -v
    """

    import json

    import pytest

    from dsv4_cc_proxy.codex.sse import translate_sse_stream


    # ---- 测试辅助 ----

    def _collect_events(chunks: list[dict]) -> list[dict]:
        """将 translate_sse_stream 异步生成器的输出收集为事件 dict 列表。

        Args:
            chunks: Chat delta dict 列表（模拟上游 SSE chunk 输入）。

        Returns:
            解析后的 SSE 事件 dict 列表，每个含 {"event": str, "data": dict}。
        """
        import asyncio

        async def _collect():
            events = []
            async for event_str in translate_sse_stream(chunks):
                lines = event_str.strip().split("\n")
                event_type = ""
                data = {}
                for line in lines:
                    if line.startswith("event: "):
                        event_type = line[7:]
                    elif line.startswith("data: "):
                        data = json.loads(line[6:])
                events.append({"event": event_type, "data": data})
            return events

        return asyncio.run(_collect())


    def _find_events(events: list[dict], event_type: str) -> list[dict]:
        """从事件列表中筛选指定类型的事件。"""
        return [e for e in events if e["event"] == event_type]
    ```

    === 测试场景 ===

    **所有场景遵循 AAA 模式：Arrange（准备输入 chunk 列表） -> Act（_collect_events） -> Assert（assert 事件类型/次序/数据）。**

    测试输入 chunk 的格式（DeepSeek Chat delta）：
    ```python
    {"choices": [{"index": 0, "delta": {"content": "Hello", "role": "assistant"}, "finish_reason": None}]}
    {"choices": [{"index": 0, "delta": {"reasoning_content": "Let me think..."}, "finish_reason": None}]}
    {"choices": [{"index": 0, "delta": {"tool_calls": [{"index": 0, "id": "call_1", "type": "function", "function": {"name": "get_weather", "arguments": ""}}]}, "finish_reason": None}]}
    {"choices": [{"index": 0, "delta": {"tool_calls": [{"index": 0, "function": {"arguments": "{\"loc"}}]}, "finish_reason": None}]}
    {"choices": [{"index": 0, "delta": {}, "finish_reason": "stop"}]}
    ```

    === 场景 1: test_text_stream (CODX-05) ===
    - 纯文本 delta.content 流（无 reasoning，无 tool_calls）
    - 输入：3 个 delta.content chunks + finish_reason
    - 验证：
      1. 首个事件是 `response.created`
      2. 第二个事件是 `response.in_progress`
      3. 有 1 个 `response.output_item.added` 且 item.type == "message"
      4. 有 1 个 `response.content_part.added`
      5. 每个 content delta 对应 1 个 `response.output_text.delta`，delta 值与输入匹配
      6. 最后有 `response.output_item.done` 且 item.content[0].text 包含完整累计文本
      7. 最后有 `response.completed`

    === 场景 2: test_reasoning_stream (CODX-13) ===
    - 纯 reasoning_content 流（无 text，无 tool_calls）
    - 输入：2 个 reasoning_content chunks + finish_reason
    - 验证：
      1. 首个 output_item.added 的 item.type 是 "reasoning"
      2. 有 reasoning_text.delta 事件，delta 值与输入匹配
      3. output_item.done 的 item.type 是 "reasoning"
      4. 无 output_text.delta 事件

    === 场景 3: test_reasoning_to_text_transition (CODX-15) ===
    - reasoning -> text 的类型转换
    - 输入：reasoning_content chunks -> content chunks -> finish_reason
    - 验证：
      1. 首个 output_item.added(item.type == "reasoning")
      2. reasoning_text.delta 事件
      3. reasoning 的 output_item.done(event: response.output_item.done, item.type == "reasoning")
      4. 新 output_item.added(item.type == "message") + content_part.added
      5. output_text.delta 事件
      6. 最终 text 的 output_item.done(含 content[] 累计文本)

    === 场景 4: test_tool_call_stream (CODX-08) ===
    - 单个工具调用流
    - 输入：tool_calls delta（含 id+name） -> 2 个 arguments delta -> finish_reason
    - 验证：
      1. output_item.added 的 item.type == "function_call"，含 name 和 call_id
      2. function_call_arguments.delta 事件，delta 值匹配累计 arguments
      3. function_call_arguments.done 事件
      4. output_item.done 的 item.arguments 含完整累计 arguments

    === 场景 5: test_parallel_tool_calls (CODX-09) ===
    - 多工具并行调用（index 0 和 index 1 交错）
    - 输入：tool_calls delta 同时含 index 0 和 index 1 的 arguments 片段
    - 验证：
      1. 首个 chunk（id+name for index 0+1）触发 2 个 output_item.added(event 事件)
      2. 后续 arguments delta 分发到对应 index 的 function_call_arguments.delta
      3. finish_reason 时 2 个工具各自有独立的 output_item.done
      4. 各工具的 arguments 独立累计，无交叉污染

    === 场景 6: test_text_to_tool_transition (CODX-15) ===
    - 文本完成后发起工具调用
    - 输入：content chunks -> tool_calls delta -> finish_reason=tool_calls
    - 验证：
      1. text output_item.added + output_text.delta 正常
      2. tool_calls 出现时 text 的 output_item.done 先发出
      3. function_call 的 output_item.added 随后发出
      4. function_call_arguments.delta 正常
      5. finish_reason 时 function_call 的 output_item.done 正常

    === 场景 7: test_full_lifecycle (CODX-06) ===
    - 完整混合生命周期：reasoning -> text -> tool_calls -> finish_reason
    - 输入：reasoning -> content -> tool_calls -> finish_reason
    - 验证：
      1. 事件序列精确匹配：created -> in_progress -> output_item.added(reasoning) -> reasoning_text.delta -> output_item.done(reasoning) -> output_item.added(message) -> content_part.added -> output_text.delta -> output_item.done(text) -> output_item.added(function_call) -> function_call_arguments.delta -> function_call_arguments.done -> output_item.done(function_call) -> completed
      2. 事件顺序严格等于预期序列（逐个 assert 事件类型）

    === 场景 8: test_edge_cases ===

    - 子测试 a: 空流（空 list）-> 只有 created + in_progress + completed，无其他事件
