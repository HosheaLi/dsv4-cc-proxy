<action>
创建 tests/test_main.py，使用 pytest   monkeypatch + unittest.mock.patch，覆盖 __main__.py 所有执行路径。

测试模式（来自 RESEARCH.md + PATTERNS.md）:
- monkeypatch.setattr("sys.argv", [...]) 设置 CLI 参数
- tmp_path 创建临时 PID 文件
- patch("os.kill") 避免实际发信号
- patch("uvicorn.run") 避免实际启动服务器
- patch("dsv4_cc_proxy.__main__.os.kill") 和 patch("dsv4_cc_proxy.__main__.os.unlink") 隔离文件系统操作
- 用 from dsv4_cc_proxy.__main__ import main 测试 main() 入口

必须覆盖的测试场景（每个场景一个函数，遵循 test_场景_预期 命名）:

1. test_stop_pidfile_not_found: --stop 时 PID 文件不存在 -> sys.exit(1)
   - monkeypatch sys.argv + 不存在 pidfile -> pytest.raises(SystemExit) assert code==1

2. test_stop_normal_sigterm: --stop 正常流程 -> SIGTERM 发送 -> PID 文件清理
   - tmp_path pidfile.write_text("99999") + monkeypatch sys.argv + patch("os.kill") -> mock_kill.assert_called_once_with(99999, signal.SIGTERM) + assert not pidfile.exists()

3. test_stop_already_dead: --stop 时进程已不存在（ProcessLookupError）-> 清理 PID 文件后返回
   - patch("os.kill", side_effect=ProcessLookupError()) -> 不崩溃, pidfile 清理

4. test_stop_graceful_timeout: --stop 时 SIGTERM 超时(5s 未退出) -> SIGKILL 强制杀
   - 模拟 os.kill 前几次成功, 最后 ProcessLookupError 表示 SIGKILL 后进程消失 -> 清理 pidfile
   - 可通过 patch("time.sleep") 跳过 5s 等待

5. test_main_already_running: 启动时 PID 文件存在且进程存活 -> sys.exit(1)
   - tmp_path pidfile.write_text(str(os.getpid())) + monkeypatch sys.argv + "os.kill(pid, 0)" 成功 -> pytest.raises(SystemExit)

6. test_main_stale_pidfile: 启动时 PID 文件存在但进程已死 -> 清理后继续启动
   - tmp_path pidfile + patch("os.kill", side_effect=ProcessLookupError()) -> pidfile 被清理, uvicorn 被调用

7. test_main_startup: 正常启动 -> uvicorn.run 被正确调用
   - monkeypatch sys.argv + patch("uvicorn.run") -> uvicorn.run.assert_called_once(), pidfile 创建

8. test_main_startup_with_dump: DUMP_DIR 启用时启动 -> 打印 DUMP 警告
   - monkeypatch.setenv("PROXY_DUMP_DIR", "/tmp/dump") + reload + uvicorn run -> 不崩溃

9. test_version_importable: VERSION 可导入且为有效字符串
   - from dsv4_cc_proxy._version import VERSION; assert VERSION is not None; assert isinstance(VERSION, str); assert len(VERSION) > 0
   - (版本无关验证: 不硬编码特定版本号, 避免 Plan 04 版本升级后的回归冲突)

10. test_stop_pidfile_corrupted: --stop 时 PID 文件内容不是整数 -> 合理错误处理
    - pidfile.write_text("not_a_number") + 验证不崩溃

**重要约束:**
- 所有使用 `__main__.py` 的内存的测试必须在函数内部 `import`（延迟导入），避免 import side effects
- tests/test_main.py 级别不要 import __main__（会触发 import side effects）
- 使用 `from dsv4_cc_proxy.__main__ import _stop, main` 时注意 _stop 使用默认 PIDFILE_DEFAULT，测试中需要用 monkeypatch 覆盖
- 每条测试独立，不依赖共享状态
- 文件头 docstring 说明用途和运行命令
- 遵循现有 test_proxy.py 的纯函数 AAA 模式
