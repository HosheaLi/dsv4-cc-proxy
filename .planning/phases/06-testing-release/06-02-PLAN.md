**必须覆盖的 4 个 E2E 场景:**

### 类 1: TestNonStreamE2E
验证完整非流式 Codex 请求 -> DeepSeek Chat -> Responses API JSON 响应。

- test_non_stream_json_response: 发送 stream=false 请求, mock 上游返回 Chat JSON, 验证返回 Responses API JSON 结构
  - mock_data: {"id":"chat-e2e","choices":[{"message":{"content":"Hello from DeepSeek"}}],"usage":{"input_tokens":5,"output_tokens":3,"total_tokens":8}}
  - 验证: status=200, body["object"]=="response", body["status"]=="completed", output[0].type=="message", output[0].content[0].text=="Hello from DeepSeek"

### 类 2: TestStreamE2E
验证完整流式 Codex 请求 -> DeepSeek Chat SSE -> Responses API SSE 事件流。

- test_stream_sse_events: 发送 stream=true 请求, mock 上游返回 SSE chunks, 验证返回 SSE 事件流包含 lifecycle 事件
  - mock SSE chunks 包含最小 Chat completion chunk: `data: {id:"...", choices:[{delta:{content:"Hi"}, finish_reason:"stop"}]}`
  - 验证: status=200, content-type=text/event-stream, "response.created" in body, "response.completed" in body

### 类 3: TestCompactE2E
- test_compact_returns_501: POST /v1/responses/compact -> 501 + JSON error body
  - 验证: status=501, body["error"]["type"]=="not_supported", body["error"]["code"]=="501"

### 类 4: TestAuthE2E
- test_auth_header_passthrough: 发送带 Authorization header 的 Codex 请求, 验证透传到上游
  - 与 test_responses.py:TestAuthPassthrough.test_auth_header_passthrough 模式一致
  - 验证: build_request call_args 中 headers.authorization 等于发送的值

**重要约束:**
- 所有上游请求使用 mock（`patch("dsv4_cc_proxy.proxy._get_client")`）, 不连接真实 API
- mock 数据使用最小够用数据, 不创建复杂 fixture
- 遵循 tests/test_responses.py 的 TestClient + patch 模式
- 文件独立可运行: `python -m pytest tests/test_e2e.py -v`
- 与 test_responses.py 的测试不共享 mutable 状态
