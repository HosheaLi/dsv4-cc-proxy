    ### 2. 辅助函数（全部 _ 前缀，纯函数，无 class）

    #### _extract_content_text(content: str | list | None) -> str | None

    D-07 的精确实现：
    - content 为 None → 返回 None
    - content 为字符串 → 直接返回字符串
    - content 为数组 → 过滤 type == "input_text" 的块，提取 text 字段，用 "\n" 拼接
    - 非预期类型 → 返回 None（不崩溃）
    - 使用 isinstance() 检查类型（PATTERNS.md 模式）

    #### _merge_system_messages(instructions: str | None, developer_messages: list[dict]) -> dict | None

    D-05 的精确实现：
    - 如果 instructions 存在且非空 → 追加到 parts 列表
    - 遍历 developer_messages，对每个消息调用 _extract_content_text(msg.get("content"))
    - 如果提取的内容非空 → 追加到 parts 列表
    - 如果 parts 为空 → 返回 None（不生成空 system 消息，Pitfall 4）
    - 否则返回 {"role": "system", "content": "\n\n".join(parts)}

    #### _translate_input_items(input_array: list) -> list[dict]

    D-08 的分发逻辑（用于 dispatch）：
    - 初始化 messages = [], pending_reasoning = []
    - 对 input_array 中的每个 item：
      - item_type = item.get("type", "") 或 item.get("role", "")
      - 如果 type == "message" → 处理 role 分发:
        - role == "developer" → 跳过（已在外部处理）
        - role == "user" → 创建 {"role": "user", "content": _extract_content_text(item.get("content"))}
        - role == "assistant" → 创建 {"role": "assistant", "content": _extract_content_text(item.get("content"))}
      - 如果 type == "function_call" → D-06:
        - 查找 messages 中最后一个 role == "assistant" 的索引
        - 如果存在 → 在该消息的 tool_calls 列表中追加新条目
        - 如果不存在 → 创建合成 assistant 消息并追加到 messages:
          {"role": "assistant", "content": None, "tool_calls": []}
        - tool_calls 条目结构: {"id": item["id"], "type": "function", "function": {"name": item["name"], "arguments": item["arguments"]}}
        - 注意：此 task 不处理 tool 定义格式转换（type: "function" 包装等）— 那是 Phase 3。
          此处只做结构性的 tool_calls 附加（添加到前一条 assistant 消息）。
      - 如果 type == "function_call_output" → 创建 tool role 消息:
        {"role": "tool", "tool_call_id": item["call_id"], "content": item["output"]}
      - 如果 type == "reasoning" → D-09, D-10:
        - 从 item.get("content", []) 中提取所有 type == "reasoning_text" 的 text，用 "\n" 拼接
        - 从 item.get("summary", []) 中提取所有 type == "summary_text" 的 text，用 "\n" 拼接
        - 如果有 summary 但无 reasoning_text → 用 summary
        - 将结果追加到 pending_reasoning 列表
      - 否则 → D-08: logger.warning("[CODEX] unknown input item type: %s, skipping", item_type) 并跳过

    - 翻译完所有 items 后，处理 pending_reasoning:
      - 如果 pending_reasoning 非空:
        - 查找最后一个 role == "assistant" 的消息
        - 如果存在 → 将 reasoning 内容注入其 reasoning_content 字段（多个用 "\n" 拼接已在前一步骤完成，此处拼接 pending_reasoning）
        - 如果不存在后续 assistant → D-09: 不注入（pending_reasoning 丢弃）
        - D-13: 如果最后一个消息不是 assistant（例如 user）→ logger.warning + 丢弃 pending_reasoning，不做结构性修复

    - 返回 messages 列表

    #### _ensure_reasoning_content(messages: list[dict]) -> None

    D-11 的精确实现：
    - 遍历 messages
    - 跳过非 role == "assistant" 的消息
    - 检查消息: 有 "tool_calls" 键且无 "reasoning_content" 键
    - 如果满足条件 → msg["reasoning_content"] = ""
    - logger.debug("[CODEX] injected reasoning_content: '' on assistant with tool_calls")

    ### 3. 主入口函数 translate_request

    def translate_request(request_body: dict) -> dict:
        """Responses API 请求体 → Chat Completions 请求体。

        Pure function: 不修改输入（D-03 强制 deepcopy）。
        返回全新字典。

        Processing:
        1. Deep copy input
        2. Extract `instructions` and `input` items
        3. Separate developer role messages
        4. Translate non-developer input items to Chat messages
        5. Apply pending reasoning if applicable
        6. Ensure reasoning_content on assistant with tool_calls
        7. Merge system messages (instructions + developer) and prepend
        8. Resolve model via resolve_model()
        9. Rename max_output_tokens → max_tokens
        10. Remove Responses-only fields: include
        11. Set messages in output body
        12. Return output
        """

    ### 4. translate_request 内部精确处理步骤

    ```
    输入: request_body (Responses API 格式)
    输出: 全新 dict (Chat Completions 格式)

    body = copy.deepcopy(request_body)

    # 1. 提取并移除 Responses 独有字段
    instructions = body.pop("instructions", None)
    input_array = body.pop("input", [])

    # 2. 分离 developer 消息
    developer_msgs = []
    other_items = []
    for item in input_array:
        if (isinstance(item, dict)
            and item.get("type") == "message"
            and item.get("role") == "developer"):
            developer_msgs.append(item)
        else:
            other_items.append(item)

    # 3. 翻译 items
    messages = _translate_input_items(other_items)

    # 4. 后处理: 确保有 tool_calls 的 assistant 包含 reasoning_content
    _ensure_reasoning_content(messages)

    # 5. 合并系统消息
    system_msg = _merge_system_messages(instructions, developer_msgs)
    if system_msg is not None:
        messages.insert(0, system_msg)

    # 6. 解析模型
    original_model = body.get("model", "")
    body["model"] = resolve_model(original_model) if original_model else CODEX_DEFAULT_MODEL

    # 7. 处理 max_output_tokens → max_tokens
    if "max_output_tokens" in body:
        body["max_tokens"] = body.pop("max_output_tokens")

    # 8. 移除 Responses-only 字段
    body.pop("include", None)

    # 9. 设置 messages
    body["messages"] = messages

    return body
    ```

    ## 注意事项

    ### D-12（推迟到 Phase 4）
    - 不要在 Phase 2 中实现 reasoning → thinking 参数映射
    - 保留 input 中的 model 字段但不要添加 thinking 参数

    ### D-13（异常序列处理）
    - reasoning → user 等异常序列只发 WARNING 跳过 reasoning
    - 不做结构性修复（如移除 user 消息等）

    ### Content: None vs Content: ""
    - assistant 消息的 content 可为 None（D-06 规定）
    - 如果 DeepSeek 对 content: None 报错，使用 content: "" 替代
    - 在 docstring 中注释此注意事项

    ### 模型解析回退
    - 如果 input body 不含 model 字段，使用 CODEX_DEFAULT_MODEL（从 config.py 导入）

    ### 日志前缀
    - 所有日志消息使用 "[CODEX]" 前缀（与 config.py 一致）
    - logger.info 用于结构性变更（如 system 消息合并）
    - logger.warning 用于可恢复问题（未知类型、异常序列）
    - logger.debug 用于详细跟踪（reasoning_content 注入等）
