    ### 测试用例清单（每个必须实现）

    **组 1: 基础消息翻译**

    1. test_simple_user_message
       - input: `[{"role": "user", "content": "Hello"}]`, instructions: "Be helpful"
       - 验证: system 消息置顶（content = "Be helpful"），user 消息紧随其后

    2. test_string_content
       - input: `[{"role": "user", "content": "plain string"}]`
       - 验证: user 消息 content 为 "plain string"（字符串形式直接通过）

    3. test_message_content_array
       - input: user 的 content 为 `[{"type": "input_text", "text": "Part1"}, {"type": "input_text", "text": "Part2"}]`
       - 验证: 翻译后 content = "Part1\nPart2"（input_text 类型拼接）

    4. test_message_content_none
       - input: user 的 content 为 None
       - 验证: user 消息 content 为 None（透传）

    5. test_assistant_message
       - input: `[{"role": "assistant", "content": "I think so"}]`
       - 验证: 翻译后 role = "assistant", content = "I think so"

    6. test_no_input_empty
       - input: `[]`, 无 instructions
       - 验证: messages = []（无 system 消息、无 user 消息）

    **组 2: System 消息合并**

    7. test_merge_instructions_and_developer (CODX-04)
       - input: instructions = "Be concise", developer messages: `[{"role": "developer", "content": "Use Python"}]`
       - 验证: system 消息 = "Be concise\n\nUse Python"（\n\n 分隔），且 developer 消息不出现在翻译后的 messages 中

    8. test_no_system_when_empty (CODX-04, Pitfall 4)
       - input: instructions = None，无 developer 消息
       - 验证: messages 中无 system 消息（不生成空 system）

    9. test_instructions_only_no_developer
       - input: instructions = "Helpful", 无 developer 消息
       - 验证: system 消息 content = "Helpful"（仅 instructions）

    10. test_developer_only_no_instructions
        - input: 无 instructions, developer: `[{"role": "developer", "content": "Be Pythonic"}]`
        - 验证: system 消息 content = "Be Pythonic"（仅 developer）

    **组 3: 工具调用翻译**

    11. test_function_call_to_tool_calls (CODX-11)
        - input: user 消息后跟 function_call: `{"type": "function_call", "id": "call_1", "name": "Bash", "arguments": "{}", "status": "completed"}`
        - 验证: assistant 消息包含 tool_calls 列表，其中第一个条目 id = "call_1"

    12. test_synthetic_assistant (CODX-11, D-06)
        - input: function_call 是第一个（也是唯一一个）item，前面无 assistant 消息
        - 验证: 创建合成 assistant 消息（role="assistant", content=None, tool_calls=[...]）

    13. test_multiple_function_calls
        - input: 一个 assistant 消息后跟两个连续的 function_call
        - 验证: 两个 tool_calls 都附加到同一个 assistant 消息上

    14. test_function_call_output_to_tool (CODX-11)
        - input: function_call_output: `{"type": "function_call_output", "call_id": "call_1", "output": "file content"}`
        - 验证: 翻译为 role="tool", tool_call_id="call_1", content="file content"

    **组 4: Reasoning 处理**

    15. test_reasoning_folds_to_next_assistant (CODX-14, D-09, D-10)
        - input: user 消息 → reasoning item (content="I need to think") → assistant 消息
        - reasoning item: `{"type": "reasoning", "id": "rs_1", "content": [{"type": "reasoning_text", "text": "Let me analyze"}]}`
        - 验证: assistant 消息包含 reasoning_content="Let me analyze"

    16. test_reasoning_content_extraction
        - input: reasoning item 含多个 reasoning_text 块
        - 验证: reasoning 内容用 "\n" 拼接后注入 assistant

    17. test_inject_reasoning_content (CODX-14, D-11)
        - input: assistant 有 tool_calls 但无 reasoning_content
        - 验证: 翻译后 assistant 包含 reasoning_content=""

    18. test_reasoning_no_following_assistant (CODX-14, D-09)
        - input: 末尾有 reasoning 但无后续 assistant 消息
        - 验证: 不抛出异常，reasoning 内容不注入（丢弃）

    19. test_reasoning_anomalous_sequence (D-13)
        - input: reasoning → user 消息序列
        - 验证: 不抛出异常，reasoning 内容跳过，user 消息正常翻译

    **组 5: 边界情况**

    20. test_unknown_type_skipped (D-08)
        - input: 包含未知 type（如 "unknown_type_xyz"）的 item
        - 验证: 该 item 被跳过，不影响其他 items 的翻译

    21. test_translate_request_immutable
        - input: 含 function_call 的请求体
        - 验证: 调用 translate_request 后，输入 dict 与 deepcopy 快照完全相同

    **组 6: 模型解析**

    22. test_model_resolved_via_config
        - 使用 monkeypatch.setenv + reload 设置 CODEX_DEFAULT_MODEL 和 CODEX_MODEL_MAP
        - input: `{"model": "gpt-5.3-codex", "input": [{"role": "user", "content": "Hi"}]}`
        - 验证: 输出 model = 映射后的 DeepSeek 模型名

    23. test_max_output_tokens_renamed
        - input: `{"max_output_tokens": 8192, "input": [...]}`
        - 验证: 输出包含 "max_tokens": 8192，不含 "max_output_tokens"

    ## 注意事项

    ### Testing patterns from existing tests

    - **无 monkeypatch 的测试**（大多数测试）→ 直接调用 codex_translate.translate_request()，不需 reload
    - **有 monkeypatch 的测试**（test_model_resolved_via_config）→ monkeypatch.setenv + reload(codex_translate)
    - **不可变性测试** → 用 copy.deepcopy 创建输入快照，调用函数后比较
    - **断言风格** → 针对特定字段断言（不比较整个 dict，except for simple cases）

    ### 测试数据构造

    使用内联 dict 构造测试数据（与 test_codex.py 和 test_proxy.py 风格一致）。

    示例简单用户消息:
    ```python
    body = {
        "model": "test-model",
        "input": [
            {"role": "user", "content": "Hello"}
        ],
        "instructions": "Be helpful.",
    }
    result = codex_translate.translate_request(body)
    ```

    ### 测试验证策略

    每个测试验证翻译结果的核心字段，而非完整字典比较：
    - assert result["messages"][N]["role"] == "expected_role"
    - assert result["messages"][N]["content"] == "expected_content"
    - assert "tool_calls" in result["messages"][N]
    - assert result["messages"][N]["tool_calls"][0]["id"] == "call_id"
    - assert result["messages"][N].get("reasoning_content") == ""
    - assert result["model"] == "deepseek-v4-flash" (env var dependent)

    ### 不测试的内容

    - 不测试 Phase 4（SSE 流处理）的功能
    - 不测试 tool 定义格式转换（Phase 3 范围）
    - 不测试 HTTP/路由功能（Phase 5 范围）
